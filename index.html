Eventize

A jQuery-based plugin for easily tracking events via Google Analytics

Easily track clickable or hoverable events via Google Analytics assigned to any "eventize(d)" location. The basic example would be to simply include a linked item within an unordered list and then call $(".eventize").eventize() on that list. Any item with the class "track-click" or "track-hover" within the Eventize block will be monitored. 


<ul class="eventize">
   <li>
      <a href="#" data-action="Track This Event"
          data-category="Trackable Event"
          data-label="User Clicked This"
          class="track-click">Click Me</a>
   </li>
</ul>

Parameters

There really isn't much to it after that. Eventize a container and then add a class of "track-click" or "track-hover" to items within the container. From there, you just need to declare the following attributes and Eventize will automagically pick them up.

  • data-action
  • data-category
  • data-label
  • data-url (Optional)

If you are unfamiliar with Actions, Categories and Labels, I suggest reading through Google Analytics definitions.

data-url

The data-url parameter is optional, but can be used to easily send a user to another location once Eventize has tracked the event. A good examle of this would be to use a non-anchored element (i.e. <em> vs. <a>) for tracking an event, subsequently sending the user to a new page or a "mailto:" option. For reasons that should be obvious, it is not recommended to append the data-url parameter to "track-hover" locations.

Debug Mode

There are definitely times that you want to see what is going to be sent prior to hooking Eventize up to Google Analytics. In these cases, Eventize has been configured with the option "debug". Currently this is the only option that Eventize supports, but debug mode will allow you to output the data that you are passing into Eventize to the console for review. This is obviously helpful for debugging without sending data to Google Analytics, etc. This can be accomplished by setting the debug option to true: 


   $(".eventize").eventize({ debug: true });

Use with Font Awesome Social Icons

I love Font Awesome, so I use it all of the time. I have embedded it within this project for that reason. If you want to use Eventize without Font Awesome, you are more than welcome to do so. To remove it from this project, simply delete the /eventize/font directory and delete the CSS below the comment stating the beginning of Font Awesome styles from within the eventize.css file.

If you want to use Font Awesome, as I mentioned, I have already built into this project's CSS file and have included the /font directory to gain access to the iconography. So, you don't really have to do anything!

Also, since I use social icons a lot, I have included the basic social colors as part of the Eventize CSS file. Here is an example of how to use Font Awesome and the social icons.
